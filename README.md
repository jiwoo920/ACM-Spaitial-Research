# Evaluating Spatial Feasibility in LLM-Generated Household Evacuation Decisions

This repository contains a reproducible research package for an ACM SIGSPATIAL Student Research Competition extended abstract.

The study evaluates whether large language model (LLM) generated household evacuation decisions remain feasible when assessed against household transportation constraints and lightweight spatial context. The main result is not that spatially grounded prompting solves evacuation planning. Instead, the evaluation shows that fluent LLM-generated evacuation plans can preserve feasibility risks, especially for no-vehicle households.

## Data

The tract profile file contains selected official ACS 2023 5-year tract-level variables for Los Angeles County, including median household income, no-vehicle household rate, poverty rate, household size, children share, elderly share, and public transportation commute share.

The household personas are synthetic. They are calibrated using official ACS tract-level variables but do not represent real households. Spatial risk and distance fields are approximate study attributes rather than a full GIS integration.

## Main Experiment

The included main experiment outputs correspond to:

- Model: GPT-4.1-mini
- Personas: 200 synthetic households
- Conditions: baseline and spatially grounded prompting
- Responses: 400 structured LLM responses
- Main output folder: `outputs/main_experiment_gpt_acs_200/`

The paper should use the included `outputs/main_experiment_gpt_acs_200/` results, not smoke-test, pipeline-validation, Ollama, or archived outputs.

## Reproduce Existing Metrics from Included Outputs

The key evaluated file is already included:

```bash
python3 scripts/analyze_results.py \
  --output-dir outputs/main_experiment_gpt_acs_200 \
  --paper-dir paper/main_experiment_gpt_acs_200 \
  --personas-file data/personas_acs_200.csv \
  --spatial-context-file data/spatial_context_acs_200.csv
```

Generate figures from the included scores:

```bash
python3 scripts/make_figures.py \
  --output-dir outputs/main_experiment_gpt_acs_200 \
  --figure-dir figures/main_experiment_gpt_acs_200
```

Validate the included analyzed outputs:

```bash
python3 scripts/validate_outputs.py \
  --output-dir outputs/main_experiment_gpt_acs_200 \
  --personas-file data/personas_acs_200.csv \
  --spatial-context-file data/spatial_context_acs_200.csv
```

## Re-run the LLM Experiment

To re-run the LLM experiment, set the required provider credential in your local shell. Do not commit credential files.

```bash
python3 scripts/run_pipeline.py \
  --provider openai \
  --model gpt-4.1-mini \
  --personas-file data/personas_acs_200.csv \
  --spatial-context-file data/spatial_context_acs_200.csv \
  --repeats 1 \
  --output-dir outputs/main_experiment_gpt_acs_200 \
  --figure-dir figures/main_experiment_gpt_acs_200 \
  --paper-dir paper/main_experiment_gpt_acs_200 \
  --max-retries 4 \
  --initial-backoff 2
```

For public repositories, avoid committing raw response files generated by a re-run unless they have been reviewed and intentionally placed in a private appendix.

## Notes on Scope

This package supports reproducibility of the SRC analysis and figures. It does not include:

- API credentials
- `.env` files
- raw LLM response files
- smoke-test outputs
- pipeline-validation outputs
- Ollama outputs
- archived working drafts or delivery zips
